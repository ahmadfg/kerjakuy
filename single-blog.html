Apa Itu Komputasi Awan?

  • Technology
  • 3 Comments

Komputasi awan adalah akses berdasarkan permintaan, melalui internet, ke sumber daya komputasi—aplikasi, server (server fisik dan server virtual), penyimpanan data, alat pengembangan, kemampuan jaringan, dan banyak lagi—yang dihosting dari jarak jauh.Pusat Data dikelola oleh penyedia layanan cloud (atau CSP). CSP menyediakan sumber daya ini dengan biaya berlangganan bulanan atau menagihnya sesuai penggunaan.

Dibandingkan dengan TI lokal tradisional, dan bergantung pada layanan cloud yang Anda pilih, komputasi awan membantu melakukan hal berikut:

  • • Biaya TI yang lebih rendah: Cloud memungkinkan Anda meringankan sebagian atau sebagian besar biaya dan upaya pembelian, pemasangan, konfigurasi, dan pengelolaan infrastruktur lokal Anda sendiri.

  • • Meningkatkan ketangkasan dan waktu untuk menilai: Dengan cloud, organisasi Anda dapat mulai menggunakan aplikasi perusahaan dalam hitungan menit, daripada menunggu berminggu-minggu atau berbulan-bulan hingga TI merespons permintaan, membeli dan mengonfigurasi perangkat keras pendukung, dan menginstal perangkat lunak. Cloud juga memungkinkan Anda memberdayakan pengguna tertentu—khususnya pengembang dan ilmuwan data—untuk membantu diri mereka sendiri dalam membuat perangkat lunak dan mendukung infrastruktur.

  • • Skalabilitas lebih mudah dan hemat biaya: Cloud memberikan elastisitas—daripada membeli kelebihan kapasitas yang tidak terpakai selama periode lambat, Anda dapat meningkatkan dan menurunkan kapasitas sebagai respons terhadap lonjakan dan penurunan lalu lintas. Anda juga dapat memanfaatkan jaringan global penyedia cloud Anda untuk menyebarkan aplikasi Anda lebih dekat dengan pengguna di seluruh dunia.

    • Istilah 'komputasi awan' juga mengacu pada teknologi yang membuat cloud berfungsi. Ini mencakup beberapa bentukinfrastruktur TI tervirtualisasi—server, perangkat lunak sistem operasi, jaringan, dan infrastruktur lainnya yang diabstraksikan, menggunakan perangkat lunak khusus, sehingga dapat dikumpulkan dan dibagi terlepas dari batasan perangkat keras fisik. Misalnya, satu server perangkat keras dapat dibagi menjadi beberapa server virtual.

    Virtualisasi memungkinkan penyedia cloud memanfaatkan sumber daya pusat data mereka secara maksimal. Tidak mengherankan, banyak perusahaan telah mengadopsi model pengiriman cloud untuk infrastruktur lokal mereka sehingga mereka dapat mewujudkan pemanfaatan maksimum dan penghematan biaya dibandingkan infrastruktur TI tradisional dan menawarkan layanan mandiri dan ketangkasan yang sama kepada pengguna akhir mereka.

    Jika Anda menggunakan komputer atau perangkat seluler di rumah atau kantor, Anda hampir pasti menggunakan beberapa bentuk komputasi awan setiap hari, baik itu aplikasi cloud seperti Google Gmail atau Salesforce, media streaming seperti Netflix, atau penyimpanan file cloud seperti Dropbox. Analis industriGartner baru-baru ini memproyeksikan bahwa belanja cloud publik pengguna akhir di seluruh dunia akan mencapai hampir USD 600 miliar pada tahun 2023.

    Jenis Komputasi Awan
  • 1. Public Cloud

  • Cloud publik adalah jenis komputasi awan di mana penyedia layanan awan membuat sumber daya komputasi—mulai dari aplikasi SaaS, hingga aplikasi individual.mesin virtual (VM), keperangkat keras komputasi bare metal, untuk melengkapi infrastruktur dan platform pengembangan tingkat perusahaan—tersedia bagi pengguna melalui internet publik. Sumber daya ini mungkin dapat diakses secara gratis, atau akses mungkin dijual berdasarkan model harga berbasis langganan atau bayar per penggunaan.

    Penyedia cloud publik memiliki, mengelola, dan memikul semua tanggung jawab atas pusat data, perangkat keras, dan infrastruktur tempat beban kerja pelanggannya dijalankan, dan biasanya menyediakan konektivitas jaringan bandwidth tinggi untuk memastikan kinerja tinggi dan akses cepat ke aplikasi dan data.

    Cloud publik adalah alingkungan multi-penyewa—Infrastruktur pusat data penyedia cloud digunakan bersama oleh semua pelanggan cloud publik. Di cloud publik terkemuka—Amazon Web Services (AWS), Google Cloud, IBM Cloud, Microsoft Azure, dan Oracle Cloud—pelanggan tersebut jumlahnya bisa mencapai jutaan.

    Banyak perusahaan yang memindahkan sebagian infrastruktur komputasi mereka ke cloud publik karena layanan cloud publik bersifat elastis dan mudah diskalakan, serta secara fleksibel dapat menyesuaikan diri untuk memenuhi tuntutan beban kerja yang terus berubah. Perusahaan lain tertarik dengan janji efisiensi yang lebih besar dan lebih sedikit sumber daya yang terbuang karena pelanggan hanya membayar sesuai penggunaan mereka. Yang lain lagi berupaya mengurangi pengeluaran untuk perangkat keras dan infrastruktur lokal.

    Cloud pribadi biasanya dihosting secara lokal di pusat data pelanggan. Namun private cloud juga dapat dihosting di infrastruktur penyedia cloud independen atau dibangun di atas infrastruktur sewaan yang ditempatkan di pusat data di luar lokasi.

    Banyak perusahaan memilih private cloud dibandingkan public cloud karena private cloud adalah cara yang lebih mudah (atau satu-satunya cara) untuk memenuhi persyaratan kepatuhan terhadap peraturan mereka. Yang lain memilih cloud pribadi karena beban kerja mereka berhubungan dengan dokumen rahasia, kekayaan intelektual, informasi pengidentifikasi pribadi (PII), catatan medis, data keuangan, atau data sensitif lainnya.

    Dengan membangun arsitektur cloud pribadi menurutawan asli prinsipnya, suatu organisasi memberikan fleksibilitas untuk dengan mudah memindahkan beban kerja ke cloud publik atau menjalankannya dalam aawan hibrida (lihat di bawah) lingkungan kapan pun mereka siap.

  • 2. Hybrid Cloud

  • Cloud hibrid adalah seperti yang terlihat—kombinasi lingkungan cloud publik dan pribadi. Secara khusus, dan idealnya, cloud hybrid menghubungkan layanan cloud pribadi dan cloud publik organisasi ke dalam satu infrastruktur yang fleksibel untuk menjalankan aplikasi dan beban kerja organisasi.

    Tujuan dari cloud hybrid adalah untuk menciptakan gabungan sumber daya cloud publik dan privat—dan dengan tingkat orkestrasi di antara keduanya—yang memberikan fleksibilitas bagi organisasi untuk memilih cloud yang optimal untuk setiap aplikasi atau beban kerja dan untuk memindahkan beban kerja secara bebas di antara keduanya. awan ketika keadaan berubah. Hal ini memungkinkan organisasi untuk memenuhi tujuan teknis dan bisnisnya dengan lebih efektif dan hemat biaya dibandingkan dengan cloud publik atau privat saja.

  • 3. Multicloud dan Hybrid Multicloud

  • Multicloud adalah penggunaan dua atau lebih cloud dari dua atau lebih penyedia cloud yang berbeda. Memiliki lingkungan multicloud bisa semudah menggunakan SaaS email dari satu vendor dan SaaS pengeditan gambar dari vendor lain. Namun ketika perusahaan berbicara tentang multicloud, mereka biasanya berbicara tentang penggunaan beberapa layanan cloud—termasuk layanan SaaS, PaaS, dan IaaS—dari dua atau lebih penyedia cloud publik terkemuka. Multicloud hibrid adalah penggunaan dua atau lebih cloud publik bersamaan dengan lingkungan cloud pribadi.

    Organisasi memilih multicloud untuk menghindari vendor lock-in, memiliki lebih banyak layanan untuk dipilih, dan mengakses lebih banyak inovasi. Namun semakin banyak cloud yang Anda gunakan—masing-masing cloud memiliki seperangkat alat manajemen, kecepatan transmisi data, dan protokol keamanannya sendiri—semakin sulit mengelola lingkungan Anda. Platform manajemen multicloud memberikan visibilitas di berbagai penyedia cloud melalui dasbor pusat, di mana tim pengembangan dapat melihat proyek dan penerapannya, tim operasi dapat mengawasi cluster dan node, dan staf keamanan siber dapat memantau ancaman.

    • Keamanan Awan

    Secara tradisional, masalah keamanan telah menjadi kendala utama bagi organisasi dalam mempertimbangkan layanan cloud, khususnya layanan cloud publik. Namun, sebagai respons terhadap permintaan, keamanan yang ditawarkan oleh penyedia layanan cloud terus melampaui solusi keamanan lokal.

    Mempertahankan keamanan cloud memerlukan prosedur dan keahlian karyawan yang berbeda dibandingkan di lingkungan TI lama. Beberapa praktik terbaik keamanan cloud meliputi hal berikut:

  • 1. Tanggung jawab bersama atas keamanan: Secara umum, penyedia cloud bertanggung jawab untuk mengamankan infrastruktur cloud dan pelanggan bertanggung jawab untuk melindungi datanya di dalam cloud—namun penting juga untuk mendefinisikan dengan jelas kepemilikan data antara pihak ketiga swasta dan publik.

  • 2. Enkripsi data: Data harus dienkripsi saat disimpan, dalam perjalanan, dan digunakan. Pelanggan perlu mempertahankan kendali penuh atas kunci keamanan dan modul keamanan perangkat keras.

  • 3. Identitas pengguna dan manajemen akses: Pelanggan dan tim TI memerlukan pemahaman dan visibilitas penuh terhadap jaringan, perangkat, aplikasi, dan akses data.

  • 4. Manajemen kolaboratif: Komunikasi yang tepat dan proses yang jelas dan dapat dipahami antara tim TI, operasi, dan keamanan akan memastikan integrasi cloud yang lancar, aman dan berkelanjutan.

  • 5. Pemantauan keamanan dan kepatuhan: Hal ini dimulai dengan memahami semua standar kepatuhan peraturan yang berlaku untuk industri Anda dan menyiapkan pemantauan aktif terhadap semua sistem yang terhubung dan layanan berbasis cloud untuk menjaga visibilitas semua pertukaran data antara lingkungan cloud publik, privat, dan hybrid.

    • Kasus Penggunaan Awan

    Dengan 25% organisasi berencana untuk pindahsemuapenerapan mereka ke cloud pada tahun depan, tampaknya kasus penggunaan komputasi awan tidak terbatas. Namun bahkan bagi perusahaan yang tidak merencanakan peralihan besar-besaran ke cloud, inisiatif tertentu dan komputasi awan merupakan perpaduan yang cocok di surga TI.

    Pemulihan bencana dan kelangsungan bisnis selalu menjadi hal yang alami bagi cloud karena cloud menyediakan redundansi yang hemat biaya untuk melindungi data dari kegagalan sistem dan jarak fisik yang diperlukan untuk memulihkan data dan aplikasi jika terjadi pemadaman listrik atau bencana lokal. Semua penyedia cloud publik besar menawarkan Disaster-Recovery-as-a-Service (DRaaS).

    Apa pun yang melibatkan penyimpanan dan pemrosesan data dalam jumlah besar dengan kecepatan tinggi—dan memerlukan kapasitas penyimpanan dan komputasi lebih besar daripada yang dapat atau ingin dibeli dan diterapkan oleh sebagian besar organisasi di lokasi—adalah target komputasi awan. Contohnya meliputi:

  • • Big Data Analytics

  • • Internet of Things (IoT)

  • • Artificial Intelligence - khususnya pembelajaran mesin Dan pembelajaran mendalam aplikasi

    • Untuk tim pengembangan yang mengadopsi Agile atau DevOps (atau DevSecOps) untuk menyederhanakan pengembangan, cloud menawarkan layanan mandiri kepada pengguna akhir sesuai permintaan yang menjaga tugas-tugas operasi—seperti menjalankan server pengembangan dan pengujian—agar tidak menjadi hambatan pengembangan.
